# CH4. FreeRTOS 포팅

## 포팅

### 포팅이란

- 실행 가능한 프로그램이 원래 설계된 바와 다른 컴퓨팅 환경( CPU, 운영체제, 서브파티 라이브러리 등 )에서 동작할 수 있도록 하는 과정
- 소스 하나로 여러 플랫폼에서 사용할 수 있는 것 == 크로스 플랫폼== One Source Multi Use

### RTOS 포팅

- 리얼 타임 커널을 마이크로 프로세서나 마이크로 컨트롤러에서 사용할 수 있도록 하는 작업
- 프로세서 의존적인 코드를 대상으로 작업
- BSP( Board Support Package )

### 포팅의 종류

- Kernel Porting
- File System Porting
- Network Porting
- USB Porting
- Bluetooth Porting
- GUI Porting
- CAN Bus Porting

### 포팅 단계

1. 사양, 포팅 참조 소스의 선례 확인 ( 가능성 타진 )
2. 차이점 분석 ( DIFF )
3. 오픈 소스에서 활용한 개발환경과 동일한 HW를 구매해서 테스트 진행
4. 대상 찾기, 포팅 시작 ( 포팅하려는 대상 이미지 시각화 )
5. 빌드 후 실행 ( 테스트)
6. crt0.S + main() 부트코드 ( 최소한의 코드 ) 준비 및 테스트
7. 타이머 & 인터럽트 포팅
8. 디렉토리 구조 생성 & 파일 배치
9. 크린 빌드 → error 0개, warning 0개
10. 디버깅
11. 커널 소스의 수정
12. 커널의 스케줄러가 동작하는 지 확인
13. 기타 기능 구현
